## 프로젝트 전체

- 이 서비스가 해결하는 문제는 무엇인가?

> 본인의 전공, 스킬, 관심분야를 기반으로 진로 코칭 

- RAG를 사용하지 않으면 어떤 문제가 생기는가?

> 할루시네이션 문제 발생

- 가장 어려웠던 구현은 무엇이고, 어떻게 해결했는가?

> 프론트엔드에서 공고별 필수 스킬이 제대로 출력되지 않는 문제 발생, 백엔드 코드 읽어가며 데이터 파이프라인 파악하고 데이터 누락 문제 찾아 해결



## 데이터 관련

- 왜 이 데이터를 선택했는가?

> 강사 제공 데이터

- 데이터의 어떤 부분이 신뢰할 수 없을 수 있는가?

> 결측치가 존재하는 부분

- SQLite와 ChromaDB에 같은 데이터를 저장하는 이유는 무엇인가?

> sqlite는 통상적인 db로써 데이터 저장용, chromadb는 벡터 기반 db로써 rag 검색용

- 결측치와 중복을 어떻게 처리했는가?

> 해당 행 제거

## RAG 관련

- RAG 구조에서 ChromaDB는 어떤 역할을 하는가?

> RAG 검색 제공

- n_results 값을 높이면 어떤 트레이드오프가 생기는가?

> 더 많은 데이터를 참고하여 품질이 향상되나 답변 생성에 더 오랜 시간 소요

- 검색 결과 품질을 어떻게 검증했는가?

> 거리 비교, 실제 입력 프롬프트와 비교.

- Hallucination이 무엇이며, RAG가 이를 어떻게 줄이는가?

> 할루시네이션은 존재하지 않는 정보를 AI가 임의로 생성하는 문제로, RAG는 실제 데이터를 참조하여 답변을 생성하게 해 환각을 줄임.

> 출처를 제공하여 답변 신뢰도를 높이기 위해

> API 서버가 정상적으로 작동중인지 확인하기 위해서


## 보안 관련

- .env 파일을 GitHub에 올리면 어떤 일이 생기는가?

> 타인이 .env에 저장된 민감정보를 도용할 수 있음.

- .env와 .env.example의 차이는 무엇인가?

> .env는 실제 환경변수 파일이고 .env.example은 예시로 활용하기 위한 예제 파일임.


## Docker 관련

- Docker를 사용하는 이유는 무엇인가?

> 사용자 환경에 구애받지 않고 일관된 품질로 서비스 제공하기 위해

- Dockerfile에서 EXPOSE와 CMD의 역할은 무엇인가?

> EXPOSE는 서비스를 노출시킬 포트 번호, CMD는 수행할 명령어
