# 나만의 AI 모델 벤치마크 문서



## 1. 문서 목적



이 문서는 CareerFit AI 프로젝트에서 사용할 AI 모델을 선택하기 위해 작성한다.



공개 리더보드 점수만 보고 모델을 고르지 않고,

내 프로젝트의 실제 사용 목적에 맞게 직접 질문하고 비교한다.



CareerFit AI의 모델 선택 기준은 다음과 같다.



* 한국어 답변 품질

* 취업·직무 상담 적합성

* 채용공고 요약 능력

* 사용자 맥락 반영 능력

* 환각이 적은 답변

* 응답 속도

* 비용 또는 무료 사용 가능성

* API 연결 난이도

* 라이선스와 사용 제한

## 3. 모델 후보 선정 기준



모델을 고를 때는 아래 항목을 먼저 확인한다.



### 3-1. 기본 확인 항목



* [ ] 내가 하려는 작업과 모델의 Task가 일치하는가?

* [ ] API로 호출 가능한 모델인가?

* [ ] 무료 티어 또는 테스트 가능한 크레딧이 있는가?

* [ ] 라이선스가 교육/포트폴리오/서비스 목적에 맞는가?

* [ ] 한국어 입력과 출력이 가능한가?

* [ ] 모델 카드에 사용법과 제한사항이 적혀 있는가?

* [ ] 최근에도 사용되거나 업데이트된 모델인가?

* [ ] 응답 속도가 실습에 적절한가?

* [ ] 비용 구조가 명확한가?



### 3-2. 라이선스 확인 항목



* [ ] License가 `apache-2.0`, `mit`, `bsd-3-clause` 등 비교적 사용하기 쉬운 라이선스인가?

* [ ] `Non-commercial`, `Research only`, `No commercial use` 문구가 있는가?

* [ ] Gated model이라 별도 승인이나 약관 동의가 필요한가?

* [ ] README 또는 Model Card에 추가 제한사항이 있는가?



### 3-3. 비용 확인 항목



* [ ] Hugging Face Inference Providers로 호출 가능한가?

* [ ] Inference Providers 가격 또는 Billing 차감 기준을 확인했는가?

* [ ] 직접 Inference Endpoint 배포가 필요한 모델인가?

* [ ] 무료 크레딧 초과 시 과금 가능성이 있는가?

* [ ] 아주 짧은 요청으로 1회 테스트 후 Usage를 확인했는가?

## 4. 평가 방식



모델 평가는 1차 정량 평가와 2차 정성 평가로 나눈다.



### 4-1. 정량 평가



각 항목을 1점부터 5점까지 평가한다.



| 점수 | 의미 |
| -- | -------- |
| 5점 | 매우 좋음 |
| 4점 | 좋음 |
| 3점 | 보통 |
| 2점 | 부족함 |
| 1점 | 사용하기 어려움 |

### 4-2. 정성 평가

점수만으로 판단하지 않고, 실제 답변을 읽고 아래 내용을 기록한다.

* 답변이 자연스러운가?

* 사용자의 전공, 경험, 목표를 반영했는가?

* 없는 정보를 지어내지 않았는가?

* 너무 일반적인 조언만 하지 않았는가?

* 실제 포트폴리오 개선에 도움이 되는가?

* 답변 형식이 보기 쉬운가?

* 한국어 표현이 어색하지 않은가?

## 6. 테스트 질문 세트

아래 질문을 모든 모델에 동일하게 입력하고 결과를 비교한다.

### 6-1. CareerFit AI 핵심 질문

```text
Python과 SQL이 필요한 데이터 분석 공고를 찾아줘.
```

```text
경영학과 학생이 지원할 수 있는 AI 관련 직무를 추천해줘.
```

```text
이 채용공고에 맞춰 포트폴리오 개선 방향을 알려줘.
```

```text
신입 기준으로 데이터 분석가 직무에 부족한 역량을 알려줘.
```

```text
다음 채용공고 내용을 5줄로 요약해줘.
```

### 6-2. 사용자 맞춤형 질문

```text
나는 AI융합교육학을 전공했고, AI 사내강사 직무에 관심이 있어. 어떤 포트폴리오를 준비하면 좋을까?
```

```text
비전공자가 FastAPI와 RAG 프로젝트를 포트폴리오로 설명하려면 어떤 구조로 말하면 좋을까?
```

```text
내 경험을 STAR 방식으로 정리해줘.
```

### 6-3. 안전성 및 환각 확인 질문

```text
채용공고에 없는 내용을 추측하지 말고, 확인 가능한 내용과 추측을 구분해서 답해줘.
```

```text
정보가 부족하면 억지로 답하지 말고, 어떤 정보가 더 필요한지 알려줘.
```

### 6-4. 형식 준수 테스트

```text
답변을 표 없이 마크다운 bullet 형식으로 정리해줘.
```

```text
초보자도 이해할 수 있게 ELI5 방식으로 설명해줘.
```
